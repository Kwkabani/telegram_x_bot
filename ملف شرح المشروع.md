# 📋 شرح المشروع الكامل: Telegram X Bot (Playwright Edition)

## 🎯 الهدف

بوت تيليجرام يسمح للمستخدم بنشر تغريدات على X (تويتر سابقاً) مع **حذف تلقائي** بعد وقت محدد، بالإضافة لدعم **التكرار** (نفس التغريدة تتكرر بفاصل زمني مع تنويع النص). وفيه **داشبورد ويب** للتحكم الكامل.

**🔄 التغيير الجذري:** تم تحويل البوت من استخدام X API (tweepy) إلى **Playwright (browser automation)** لتجنب قيود API ومنع حظر الحسابات.

---

## 🏗️ العمارة (Architecture)

```
المستخدم ← [تيليجرام] ← البوت @Moh_mediabot ← يتصل بـ [X عبر Playwright]
                                                        │
                                           [قاعدة بيانات SQLite]
                                                        │
                                           [داشبورد ويب :5000]
```

- **اللغة:** Python 3.14
- **المكتبات الأساسية:** python-telegram-bot 22.8 (ptb)، **playwright 1.60+** (تشغيل المتصفح)، FastAPI (داشبورد)، SQLAlchemy async (ORM)، APScheduler (مهام الخلفية)
- **نظام التوثيق:** **كوكيز المتصفح (cookies)** بدلاً من OAuth — المستخدم يصدر كوكيز حسابه ويرسلها للبوت مشفرة

## 🔄 تدفق العمل (Flow)

### 1. تسجيل الدخول إلى X (جديد — كوكيز)
```
[المستخدم] ← /start ← [البوت] ← يرحب ويطلب تسجيل دخول
           ← يضغط "تسجيل دخول" ← يرسل تعليمات تصدير الكوكيز
           ← يصدر الكوكيز من المتصفح (إضافة EditThisCookie)
           ← يرسل ملف JSON ← [البوت] ← Playwright يتحقق من صحة الكوكيز
           ← يستخرج معلومات الحساب (@username) ← يخزن الكوكيز (مشفرة) في DB
```

### 2. نشر منشور
```
[المستخدم] ← "منشور جديد" ← [البوت] ← يطلب النص
           ← يرسل النص ← [البوت] ← يطلب صورة (اختياري)
           ← اختيار وقت الحذف ← اختيار عدد التكرارات ← تأكيد
           ← [البوت] ← Playwright يفتح twitter.com/compose/post
           ← يكتب النص ← يرفع الصورة ← ينقر Tweet ← يحصل على ID التغريدة
           ← يحفظ في DB
           ← [Scheduler] ← بعد الوقت المحدد ← Playwright يحذف التغريدة
```

### 3. Session Keeper (جديد — إنعاش الجلسات)
```
[Scheduler] ← كل 4 ساعات ← [SessionKeeper] ← لكل مستخدم:
            ← Playwright يفتح twitter.com/home بالكوكيز
            ← إذا الجلسة سليمة ← يحفظ الكوكيز المحدّثة
            ← إذا انتهت ← يسجل تحذير (المستخدم يحتاج يعيد رفع الكوكيز)
```

### 4. الحذف التلقائي
```
[Scheduler] ← كل 60 ثانية ← [PostSweeper] ← يبحث عن منشورات
             delete_at <= now + status == published
             ← Playwright يفتح التغريدة ← ينقر Delete ← يؤكد ← يحدّث الحالة
```

### 5. التكرار
```
[Scheduler] ← كل 30 ثانية ← [ReproductionManager] ← يبحث عن
             remaining_repeats > 0 + مر وقت كافٍ (60 دقيقة)
             ← Playwright ينشر نسخة معدلة (تنويع أفضل للنص)
             ← ينقص remaining_repeats (عملية ذرية على مستوى SQL)
```

### 6. الداشبورد
```
[Admin] ← http://localhost:5000 ← يدخل كلمة السر
         ← يشوف الإحصائيات الحية من DB
         ← يعدل الرسائل ← تنعكس على البوت فوراً
         ← Pause/Resume البوت
         ← Ban/Delete/Disconnect المستخدمين
```

## 🔐 آلية الأمان (مُحسّنة)

- **الكوكيز**: مشفرة بـ `Fernet` (مكتبة cryptography) قبل التخزين في DB
- **WAL mode**: SQLite مع WAL mode لتجنب `database is locked`
- **المصادقة**: الداشبورد محمي بكلمة سر
- **المحظورين**: حقل `banned` في DB + فحص عند كل `/start`
- **الإيقاف المؤقت**: البوت يرفض أي تفاعل (إلا للأدمن) لو `bot_paused = true`
- **Anti-detection**: Playwright مع إخفاء آثار الأتمتة (webdriver, chrome.runtime, canvas)
- **Atomic operations**: عداد التكرارات على مستوى SQL وليس Python (يمنع race conditions)
- **CSRF protection**: محسّنة عبر SameSite cookies

## 📥 كيفية تصدير الكوكيز (للمستخدم)

1. افتح **Google Chrome** أو **Microsoft Edge**
2. سجل الدخول إلى **twitter.com**
3. ثبت إضافة **"EditThisCookie"** أو **"Cookie-Editor"** من متجر Chrome
4. افتح الإضافة ← اختر **Export** ← **JSON format**
5. انسخ المحتوى أو حمّله كملف
6. أرسل الملف إلى البوت

## 🧠 ملاحظات تقنية مهمة

- **Session Keeper**: البوت يشتغل كل 4 ساعات لإنعاش جلسات المستخدمين — الكوكيز ما تنتهي أبداً
- `run_polling()` بلوكينغ → الداشبورد يشتغل في `thread` منفصل (`daemon=True`)
- **تكرارات أكثر أماناً**: تنويع النص 5 صيغ مختلفة + فاصل زمني 60 دقيقة بدلاً من 5
- **Stealth mode**: Playwright مع anti-detection scripts + random delays
- **الـ Proxies**: مدعومة لكن اختيارية (يُفعل بإضافة PROXY_LIST في .env)

## 🔄 المقارنة: قبل vs بعد

| الميزة | قبل (X API) | بعد (Playwright) |
|--------|-------------|------------------|
| التوثيق | OAuth 1.0a PIN | كوكيز المتصفح |
| مكتبة X | tweepy | Playwright |
| المتطلبات | X API Keys (تطوير) | فقط متصفح |
| مدة الجلسة | حسب API | **شهور** (مع Session Keeper) |
| تنويع النص | 3 صيغ ضعيفة | 5 صيغ + عشوائية |
| فاصل التكرار | 5 دقائق | 60 دقيقة |
| Anti-detection | ❌ لا يوجد | ✅ stealth + delays |
| WAL mode | ❌ | ✅ |
| Atomic counters | ❌ Python-level | ✅ SQL-level |
